\section{Implementation and experimental verification}\label{sec:impl}

To ground the theory in empirical reality, we implemented the Apeiron category in Python using the reference implementation of Layer~1.
The script \texttt{categorical\_verification.py} performs the following steps for a given candidate observable $o$:
\begin{enumerate}
\item Recursively applies all decomposition operators to $o$ and the obtained parts, building the \textbf{closure} $\overline{\{o\}}$ -- the full subcategory generated by $o$.
\item For every object $x \neq o$ in this closure, verifies that there is \emph{no} morphism $o \to x$ (i.e., $o$ is separated).
\item Checks the multi‑axial atomicity of $o$ by querying the four decomposition operators directly.
\end{enumerate}
The script then outputs \texttt{tables/cat\_verification\_table.tex} and an overhead benchmark.

\Cref{tab:cat_verify} shows the results for four representative observables.
The Boolean, Measure, Categorical, and Information columns indicate whether the corresponding decomposition operator recognises the observable as atomic (1 = atomic, 0 = non‑atomic).
The final column reports whether the object is separated in its generated subcategory.

\subsection{Computational Overhead and Practical Feasibility}
A natural question is whether the categorical verification step introduces prohibitive runtime cost.
To assess this, we measured the time required to perform the full closure‑based separatedness test for each of the four representative observables on a standard laptop (Intel Core i7‑8750H, 2020).
Averaged over 200 repetitions, the verification times were \textbf{0.01\,ms} for \texttt{o1},
\textbf{0.20\,ms} for \texttt{o\_set\_2}, \textbf{0.01\,ms} for \texttt{o\_prime\_7},
and \textbf{0.45\,ms} for the random string \texttt{o\_random\_str}.
These numbers confirm that the categorical audit is an extremely lightweight operation; it is several orders of magnitude faster than the runtime monitoring tasks described in the APEIRON Core~\cite{Beniers2026Core}, which operate at microsecond to millisecond scale but must be executed continuously.

Crucially, the categorical verification is not part of the real‑time control loop. It is an \emph{offline} assurance step, analogous to a type‑checker or a formal proof, that can be invoked periodically or on‑demand.
The safety benefit --- a machine‑certified guarantee that the fundamental observables are indeed irreducible and that all inter‑layer translations respect composition --- far outweighs this negligible one‑time cost.
Future work will investigate caching of morphism sets and GPU‑accelerated SAT solving (for the Z3 backend) to make the verification even more efficient.
